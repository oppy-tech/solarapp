# SolarAPP+ Senior Product Engineer Interview

Welcome! We are excited to see how you think and build.

This repository contains a choice of **3 Technical Tasks**. You should choose **ONE** task that best aligns with your strengths and the role you are applying for.

## The Tasks

### [Option 1: Cloud Infrastructure](technical-interview-option-1-cloud.md)
**Focus:** AWS, ECS, IAM, Architecture.
**Goal:** Architect an asynchronous job processing system using SQS and ECS.
**Best for:** Candidates who love infrastructure-as-code, security, and system design.

### [Option 2: API Integration](technical-interview-option-2-integration.md)
**Focus:** System Design, API Design, Data Modeling.
**Goal:** Design and build a production-ready integration for "PermitTech" to programmatically onboard Installers.
**Best for:** Candidates who excel at thinking through complex integrations, edge cases, and partner APIs.

### [Option 3: Multi-Agent Engineering](technical-interview-option-3-ai-engineering.md)
**Focus:** AI Agent Orchestration, Guardrails, Verification, Full Stack.
**Goal:** Build a multi-agent workflow that executes three parallel tasks (backend, frontend, data integrity), then deliver the orchestration artifacts and output.
**Best for:** Candidates who can decompose work across AI agents, set up meaningful guardrails, and verify the results.

## Next Steps

**If you chose Option 1 or Option 2** and want to run the Laravel environment, read [**task-3-next-steps.md**](task-3-next-steps.md). Option 1 does not require it; Option 2 may use it optionally.

**If you chose Option 3,** you will use the Laravel environment *and* AI tooling — see the task instructions.

## Deliverables

1.  **Code / Artifacts:** The modified files (PHP, YAML, Blade, diagrams, etc.).
2.  **PROCESS.md:** A structured markdown file. See the requirements below.

### PROCESS.md Requirements

Your PROCESS.md should answer these specific questions:

1.  **AI Usage Log:** For each significant piece of your submission, note whether you:
    - Wrote it yourself from scratch
    - Generated it with AI and used as-is
    - Generated it with AI and then modified it (explain what you changed and why)

2.  **A Decision You Rejected:** Describe at least one approach you considered and decided against. What was it, and why did you reject it?

3.  **What Surprised You:** What was the hardest or most unexpected part of this task? What did you learn?

4.  **Code You Don't Fully Understand:** Is there any code in your submission that you couldn't confidently explain line-by-line? Be honest — this is not a trick question. We value self-awareness over false confidence.

5.  **What You'd Do Next:** If you ran out of time, what would your next steps be?

**Time Limit:** 4 Hours.
*Note: If you run out of time, that's okay! Please spend the last 15 minutes writing your PROCESS.md.*

Good luck!
